## 1. Agent 执行器状态机化重构

**问题**: 原 `AgentRuntime` 用 `isProcessing`、`pausedAgentResponse`、多处 `setTimeout(..., 100)` 拼凑流程控制,存在竞态风险,暂停/取消不能真正中断正在进行的异步操作。

**方案**: 重写为显式状态机:

```
idle → initializing → thinking →
  { awaitingApproval | awaitingQuestion | executing | waitingForUnblock }
  → idle … → completed | failed
          ↕
       paused
```

**关键改动**:
- `taskVersion` 贯穿所有异步,陈旧回调直接丢弃
- `AbortController` 贯通 AI 调用 + 命令等待
- `queueMicrotask` 替代 `setTimeout(100)` 做调度
- 事件边沿检测(`approvalResult` / `pendingInput`)避免重复触发
- pause 会 abort AI 和命令等待,resume 重入调度器
- 外部 `cancelTask` 能被识别并 abort 正在进行的异步

**文件**: `src/renderer/agent/agent-runtime.ts`, `src/renderer/components/AgentExecutor.tsx`

## 2. 命令完成判定 — Sentinel 机制

**问题**: 原来靠"输出停止增长 + shell 提示符正则"判断命令是否结束,对长时命令、交互式命令、慢网络都容易误判。

**方案**: 每条 Agent 命令被主进程包装为:

```bash
(<用户命令>); printf '\n__AGENT_DONE_<runId>__:%s\n' "$?"
```

主进程在 shell data 事件上挂等待器,命中 sentinel 正则才 resolve。

**新增 IPC**:
- `AGENT_EXEC_AWAIT`: 写入带 sentinel 的命令并等待完成
- `AGENT_CANCEL_EXEC`: 立刻 resolve 当前等待(reason: 'canceled')

**返回结构**: `{ output, exitCode, reason: 'done' | 'timeout' | 'canceled' | 'closed' }`

**文件**: `src/main/ipc/agent-handlers.ts`, `src/shared/constants.ts`, `src/shared/ipc-types.ts`, `src/shared/global.d.ts`, `src/main/preload.ts`

## 3. Sentinel 痕迹隐藏

**问题**: sentinel 命令的 echo 和输出标记会出现在用户终端里。

**方案**: `src/main/utils/sentinel-stripper.ts` — 流式 stripper,维护 buffer + holdback,只在确认不在匹配边界时才 flush。

**关键设计**:
- `ECHO_PATTERN`: 剥离 `); printf '\n__AGENT_DONE_...` 回显
- `MARKER_PATTERN`: 剥离 `__AGENT_DONE_...:0\n` 输出行(必须等换行到达)
- `PARTIAL_TAIL_RE`: 渐进式正则检测尾部是否正在构成 sentinel
- 不含嫌疑时立刻 flush,避免用户输入回显被延迟

**测试**: `scripts/test-sentinel-stripper.ts` — 16 个用例全过(单 chunk、跨 chunk、逐字节、背靠背命令等)

**接入点**: `ssh-handlers.ts`(用户终端)和 `agent-handlers.ts`(Agent 终端)各自创建独立 stripper 实例。

## 12. 多语言支持(i18n)

### 架构

```
src/renderer/i18n/
├── index.ts              # useI18n hook + t() 函数 + zustand store
├── types.ts              # Locale 类型
└── locales/
    ├── index.ts          # 语言包注册
    ├── zh-CN.ts          # 中文(250+ key)
    └── en-US.ts          # 英文(结构对齐,类型约束)
```

### 使用方式

```tsx
// 组件内
const { t } = useI18n();
return <span>{t('connection.status.connected')}</span>;

// 非组件代码
import { t } from '../i18n';
```

### 覆盖范围

已完成 i18n 替换的文件:
- `App.tsx` — 连接管理、标签页、状态栏、快速命令
- `AgentPet.tsx` — 智能体面板全部文本
- `AgentThinking.tsx` — 状态标签
- `AIProviderSettings.tsx` — 供应商配置
- `CommandApproval.tsx` — 命令审批弹窗
- `ConfirmDialog.tsx` — 确认/取消按钮
- `FileTransfer.tsx` — 文件传输面板
- `SettingsPanel.tsx` — 所有设置 tab + 语言切换
- `useAIStore.ts` — 错误消息
- `AgentExecutor.tsx` — 通知文案
- `agent-runtime.ts` — 步骤标题、完成原因

### 设置持久化

`AppSettings.language` 字段存到 electron-store,启动时自动加载并同步到 i18n store。

### 扩展新语言

只需新建 `locales/ja-JP.ts` + 在 `types.ts` 加 Locale + 在 `locales/index.ts` 注册。
